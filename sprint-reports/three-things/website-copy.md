## 6. Acquisition Hook — Scroll-Path Checklist

Goal: at no scroll depth is "Free for landlords" more than one screen away.

| Scroll position                    | "Free" visible? | Where                                                       |
| ---------------------------------- | --------------- | ----------------------------------------------------------- |
| Hero above-fold                    | YES             | CTA button "Start free / เริ่มใช้งานฟรี"                    |
| Sticky header (any scroll)         | YES             | "Start free" button persists in sticky nav                  |
| Feature section (1–2 screens down) | Within 1 screen | Pricing section headline immediately follows features       |
| Pricing section                    | YES             | Big headline "Free for landlords."                          |
| Trust section                      | Within 1 screen | Pricing section is one scroll above                         |
| Footer                             | YES             | "Free for landlords" note + "Start free" CTA link in footer |

**Implementation note for builder:** The sticky header must carry a "Start free" CTA that is always visible on desktop and as a full-width button on mobile. This is the single most reliable guarantor of the "never more than one screen away" rule.

## 7. Orphan-Copy Audit

No existing landing page was found in the repository. The only file at `app/(public)/` is a `.gitkeep` placeholder.

**Conclusion:** No orphaned feature promises exist in a landing page. The orphan risk lives in the auth pages (`app/(public)/login/page.tsx` and `app/(public)/signup/page.tsx`), specifically:

- `auth.beta_title`: `"🎉 Beta — Free Pro Plan for 1 Year"` — this references "Pro Plan," which is a hidden tier. Once the landing page is live, this signup-page banner should be removed or replaced with neutral beta language.
- `auth.beta_description`: `"Sign up now and get the Pro plan free for 1 year..."` — same issue. References Pro tier directly.

These are in `rental-manager/locales/en.json` lines 34–35 and mirrored in `th.json` lines 34–35. They are not in a landing page file, so they fall outside the strict audit scope, but they will create a contradiction if a visitor clicks "Start free" from the landing and lands on a signup page that advertises a Pro tier the landing never mentioned.

## Recommended Hero Variant

**Variant 3 (Landlord-first / free angle)** is the best fit for the validation goal. The five real Thai landlords being interviewed are not yet users — the first question in their mind is "what does this cost me." Variant 3 answers that before they have to ask. The subhead still names the three things (properties, payments, contracts), so no feature is hidden. Variant 1 is a close second for returning visitors who already know the price.
